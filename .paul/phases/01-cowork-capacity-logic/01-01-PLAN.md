<objective>
## Goal
Implementar lógica completa de capacidade do cowork em todo o sistema:
- **Vagas corretas:** Starter=4, Prime=4, Premium=2
- **Backend:** Validação de capacidade ao criar/editar reservas (rejeitar se lotado)
- **Calendário público:** Mostrar vagas restantes por dia + marcar indisponível quando lotado
- **Dashboard admin:** Mostrar ocupação cowork no calendário + validação no formulário

## Purpose
Garantir que o sistema de reservas do cowork é funcional e consistente em todas as interfaces — o leitor vê vagas reais, o admin gere com dados corretos, e o backend impede overbooking.

## Output
- rentalData.ts com totalSpots corretos
- database.ts com validação de capacidade
- API monthly com dados de vagas parciais (não só lotado/disponível)
- Calendário público com indicador de vagas restantes
- Dashboard calendário com resumo de ocupação cowork
- Formulário admin com validação client-side de capacidade
</objective>
