<task type="auto">
  <name>Task 1: Corrigir globo mobile nas 4 hero sections</name>
  <files>
    src/components/sections/portfolio/PortfolioHero.tsx,
    src/components/sections/rental/RentalHero.tsx,
    src/app/espaco/page.tsx,
    src/components/sections/cursos/CursosHero.tsx
  </files>
  <action>
    O problema: Em mobile, o globo usa `w-[250%]` (2.5x largura do viewport) que causa overflow horizontal.
    O container pai tem `overflow-hidden` mas o posicionamento `top-[40%]` empurra o globo para baixo,
    cortando a parte superior visível. O aspect-square combinado com a largura excessiva cria um
    elemento enorme que fica maioritariamente invisível.

    Correção em TODOS os 4 ficheiros (padrão idêntico):

    1. Alterar o container do globo de:
       ```
       <div className="absolute inset-x-0 top-[40%] md:top-[30%] lg:top-[25%] z-0 flex justify-center items-start pointer-events-none">
         <div className="relative w-[250%] md:w-[200%] max-w-[1200px] md:max-w-[2000px] lg:max-w-[2500px] aspect-square pointer-events-auto opacity-100">
       ```
       Para:
       ```
       <div className="absolute inset-x-0 top-[55%] md:top-[30%] lg:top-[25%] z-0 flex justify-center items-start pointer-events-none">
         <div className="relative w-[140%] md:w-[200%] max-w-[600px] md:max-w-[2000px] lg:max-w-[2500px] aspect-square pointer-events-auto opacity-100">
       ```

    Justificação das mudanças mobile:
    - `w-[140%]` em vez de `w-[250%]` — reduz a largura para 1.4x viewport, evitando overflow horizontal
      mas mantendo o globo grande o suficiente para criar efeito imersivo
    - `max-w-[600px]` em vez de `max-w-[1200px]` — limita o tamanho máximo em mobile
    - `top-[55%]` em vez de `top-[40%]` — posiciona o globo mais baixo para que a parte visível
      (a curvatura superior) apareça mais centrada na secção, sem ser cortada pelo topo

    NÃO alterar os valores md: e lg: (desktop mantém-se igual).
    NÃO alterar o componente Globe em si (globe.tsx).
    NÃO alterar a overlay escura nem os textos da hero.
  </action>
  <verify>
    - Abrir cada página (Portfolio, Aluguer, Espaço, Cursos) no Chrome DevTools em viewport 375x812 (iPhone)
    - Verificar que o globo é visível, centrado, sem cortes laterais
    - Verificar que não há scroll horizontal na página
    - Confirmar que em desktop (1440px) o globo mantém o aspecto original
  </verify>
  <done>AC-1 satisfied: Globo mobile centrado, sem clipping, sem overflow horizontal em todas as 4 hero sections</done>

<task type="auto">
  <name>Task 3: Corrigir calendário de estúdios cortado em mobile</name>
  <files>
    src/components/ui/StudioHourlyCalendar.tsx,
    src/components/ui/MobileBottomSheet.tsx
  </files>
  <action>
    O problema: O calendário mobile usa MobileBottomSheet (vaul Drawer) que já fornece header e
    scroll area. Mas o `calendarPanel` (motion.div) internamente tem:
    - `overflow-hidden rounded-2xl` que pode cortar o conteúdo
    - `p-4 overflow-y-auto` na área de conteúdo (duplica o scroll do MobileBottomSheet)
    - Animação `initial={{ y: 20, scale: 0.95 }}` pode causar clipping visual durante transição
    O MobileBottomSheet também tem `overflow-y-auto p-4` no content wrapper,
    criando double-scroll e padding excessivo.

    Correção em StudioHourlyCalendar.tsx:

    1. No `calendarPanel` (motion.div, linha ~141), ajustar as classes para mobile:
       Alterar de:
       ```
       className={`
         relative w-full flex flex-col overflow-hidden rounded-2xl
         ${isDesktop ? 'max-w-4xl max-h-[85vh] bg-black/95 border border-white/10 backdrop-blur-xl shadow-2xl shadow-black/50' : ''}
       `}
       ```
       Para:
       ```
       className={`
         relative w-full flex flex-col
         ${isDesktop ? 'overflow-hidden rounded-2xl max-w-4xl max-h-[85vh] bg-black/95 border border-white/10 backdrop-blur-xl shadow-2xl shadow-black/50' : ''}
       `}
       ```
       Isto move `overflow-hidden rounded-2xl` para desktop only. Em mobile, o MobileBottomSheet
       já controla overflow e border-radius.

    2. No wrapper do conteúdo interno (linha ~171), condicionar o overflow e padding para mobile:
       Alterar de:
       ```
       <div className="p-4 md:p-6 overflow-y-auto flex-1 scrollbar-thin scrollbar-thumb-white/10 scrollbar-track-transparent">
       ```
       Para:
       ```
       <div className={`flex-1 ${isDesktop ? 'p-4 md:p-6 overflow-y-auto scrollbar-thin scrollbar-thumb-white/10 scrollbar-track-transparent' : 'px-0 pt-0 pb-4'}`}>
       ```
       Em mobile: sem padding horizontal extra (MobileBottomSheet já tem p-4), sem overflow-y-auto
       duplicado, apenas pb-4 para espaçamento inferior.

    3. Na animação do calendarPanel, desativar scale em mobile para evitar clipping:
       Alterar de:
       ```
       initial={{ opacity: 0, y: 20, scale: 0.95 }}
       animate={{ opacity: 1, y: 0, scale: 1 }}
       exit={{ opacity: 0, y: 20, scale: 0.95 }}
       ```
       Para:
       ```
       initial={{ opacity: 0, y: isDesktop ? 20 : 0, scale: isDesktop ? 0.95 : 1 }}
       animate={{ opacity: 1, y: 0, scale: 1 }}
       exit={{ opacity: 0, y: isDesktop ? 20 : 0, scale: isDesktop ? 0.95 : 1 }}
       ```
       Em mobile, o MobileBottomSheet já anima a entrada — a animação dupla causa salto visual.

    NÃO alterar o layout dos time slots.
    NÃO alterar a lógica de disponibilidade/reservas.
    NÃO alterar o comportamento desktop do calendário.
  </action>
  <verify>
    - Em mobile (375px), abrir página Aluguer → selecionar estúdio → calendário abre
    - Verificar que o header de navegação de datas é 100% visível (sem corte no topo)
    - Verificar que o conteúdo scrolls suavemente dentro do bottom sheet
    - Verificar que fechar (swipe down ou X) funciona normalmente
    - Em desktop, confirmar que o calendário modal mantém o aspecto original
  </verify>
  <done>AC-3 satisfied: Calendário mobile sem clipping, scroll correto, sem sobreposição de elementos</done>
