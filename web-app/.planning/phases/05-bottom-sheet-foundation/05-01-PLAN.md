## Critical Discovery

Lenis is ALREADY disabled on mobile (LenisProvider returns bare `{children}` when isMobile is true). This means SHEET-05 requirement about "pausing Lenis" is a NO-OP on mobile — Lenis isn't running.

We only need body scroll lock (overflow: hidden on body) — vaul handles this automatically via its internal portal behavior.

## Existing Calendar Patterns

Both calendars currently:
- Detect isDesktop via `matchMedia('(min-width: 768px)')`
- Desktop: Use `createPortal` to body with fixed overlay z-[100]
- Mobile: Render inline (AvailabilityCalendar = absolute dropdown, StudioHourlyCalendar = inline panel)
- Both use Framer Motion for animations
- Both have `containerRef` for click-outside detection

We will EXTEND the mobile branch to render inside MobileBottomSheet (desktop branch remains untouched).

## vaul API

Based on research:
- `<Drawer.Root open={bool} onOpenChange={fn}>` — main wrapper
- `<Drawer.Portal>` — portal to body (automatic)
- `<Drawer.Overlay>` — backdrop
- `<Drawer.Content>` — sheet content container
- `<Drawer.Handle>` — drag handle (visual indicator)
- snapPoints prop for snap behavior
- dismissible prop for swipe-to-dismiss
- Body scroll lock is automatic when open

- Background: `bg-black/95 backdrop-blur-xl`
- Border: `border border-white/10`
- Accent: `var(--color-accent)` (#FF10F0)
- Safe areas: CSS `env(safe-area-inset-bottom)` for iOS notch/home indicator
