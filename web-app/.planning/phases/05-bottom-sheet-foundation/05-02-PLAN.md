## Integration Strategy

**Principle:** ONLY touch the mobile branch (when `!isDesktop`). Desktop branch (inside `if (isDesktop)`) is FORBIDDEN TERRITORY — do not modify a single character.

**Pattern:**
```tsx
// Desktop branch — DO NOT TOUCH
if (isDesktop) {
  return createPortal(
    <AnimatePresence>
      <motion.div className="fixed inset-0 ...">
        {calendarContent}
      </motion.div>
    </AnimatePresence>,
    document.body
  );
}

// Mobile branch — REPLACE with bottom sheet
return (
  <MobileBottomSheet
    open={true}
    onOpenChange={(open) => !open && onClose()}
    title="Selecione a Data" // or studio name
  >
    {calendarContent}
  </MobileBottomSheet>
);
```

**Why this works:**
- Desktop code path unchanged → zero regression risk
- Mobile gets bottom sheet wrapper → new UX
- Calendar content (`calendarContent` / `calendarPanel`) reused → no duplication
- `onClose` callback already exists in both components

## Styling Adjustments

When moving content into MobileBottomSheet:

1. **Remove mobile-specific positioning:**
   - AvailabilityCalendar: Remove `absolute top-full left-0 mt-2` classes
   - StudioHourlyCalendar: Already inline, no positioning to remove

2. **Keep content styling:**
   - Background, border, padding, rounded corners stay the same
   - MobileBottomSheet provides the sheet container, calendar provides content

3. **Width adjustments:**
   - AvailabilityCalendar mobile: Change from `w-[320px]` to `w-full` (fill sheet width)
   - StudioHourlyCalendar: Already `w-full`, no change needed

4. **Animation adjustments:**
   - Remove `initial/animate/exit` props from calendar content motion.div in mobile branch (MobileBottomSheet handles sheet animation)
   - Keep AnimatePresence for internal content transitions (month/day changes)

## Close Button Behavior

Both calendars already have:
- `onClose` prop callback
- Close on click outside (via containerRef + mousedown listener)
- Close on Escape key

With bottom sheet:
- Click outside → vaul's dismissible prop handles this
- Escape key → keep existing handler (still needed)
- Drag down → vaul handles via swipe dismiss
- Close button in header → call onClose

**Remove click-outside listener in mobile branch** (vaul handles it):
```tsx
useEffect(() => {
  if (isDesktop) {
    // click outside handler
  }
  // Skip in mobile — vaul handles dismissible behavior
}, [isDesktop, onClose]);
```

## Parent Components

Parent components (GearRenting, CoworkStandalone, StudioRenting) DO NOT need changes:
- They set `calendarOpen` state (true/false)
- They pass `onClose={() => setCalendarOpen(false)}`
- This works identically with bottom sheet — when sheet closes (via swipe/click/escape), `onClose` is called, state updates, sheet unmounts

No parent component changes needed in this plan.
