  <action>
1. Install dependencies: `npm install splitting react-wrap-balancer`

2. Create `src/components/animations/CharReveal.tsx` — a character-by-character reveal component:
   - Props: `text: string`, `tag?: 'h2' | 'h3' | 'span'` (default 'h2'), `className?: string`, `staggerDelay?: number` (default 0.03), `duration?: number` (default 0.6)
   - Use `useRef` for container, `useReducedMotion` hook, and `useInView(containerRef, { triggerOnce: true })` from `@/hooks/useInView`
   - On mount (useEffect), dynamically import Splitting.js: `const Splitting = (await import('splitting')).default; await import('splitting/dist/splitting-cells.css');`
   - Call `Splitting({ target: el, by: 'chars' })` to split text into individual char spans
   - When `isInView` becomes true AND `prefersReducedMotion` is false: animate each `.char` with GSAP stagger — `gsap.fromTo(chars, { opacity: 0, y: 20 }, { opacity: 1, y: 0, duration, stagger: staggerDelay, ease: 'power3.out' })`
   - When `prefersReducedMotion` is true: set all chars to opacity 1 immediately (no animation) — TEXT-04 requirement
   - Add library ownership comment at top: `// Animation: GSAP (scroll-triggered character reveal — see ANIMATION_LIBRARY_GUIDE.md)`
   - Cleanup: kill GSAP tweens on unmount via return in useEffect
   - Render the tag element with `ref`, `className`, and `text` as children (Splitting.js modifies DOM in-place)

IMPORTANT: Splitting.js is a DOM manipulation library — it must run client-side only. The dynamic import prevents SSR issues. Do NOT import at module level.
IMPORTANT: Use `useInView` from `@/hooks/useInView` (IntersectionObserver-based), NOT framer-motion useInView. This is a GSAP animation — per LIBRARY-01 decision, GSAP components use native observers.
  </action>
