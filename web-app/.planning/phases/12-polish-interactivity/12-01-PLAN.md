Create a `ParallaxImage` component that wraps a Next.js Image with GSAP ScrollTrigger-driven parallax.

**Props interface:**
```typescript
interface ParallaxImageProps {
  src: string;
  alt: string;
  className?: string;        // Container div classes
  imageClassName?: string;   // Next.js Image classes
  aspectRatio?: string;      // Tailwind aspect class, default "aspect-video"
  intensity?: number;        // Parallax intensity 0-15 (percentage), default 10
  children?: ReactNode;      // Overlay content (gradients, titles)
  layoutId?: string;         // Framer Motion layoutId for lightbox integration
  onClick?: () => void;      // Click handler for lightbox trigger
}
```

**Implementation details:**
- Use `useRef` for container and image wrapper.
- Use `useReducedMotion` hook from `@/hooks/useReducedMotion`.
- Detect mobile with `useState` + `useEffect` checking `window.matchMedia('(max-width: 767px)')` — listen for changes.
- If mobile OR reducedMotion: render plain image with NO parallax (POLISH-04 compliance).
- If desktop + motion allowed: use `useGSAP` with config object format `{ scope: containerRef }` (REACT19-01 decision).
- Inside `useGSAP`: create `gsap.fromTo` on the image wrapper with `yPercent: -intensity` to `yPercent: intensity`, tied to a `ScrollTrigger` with `trigger: containerRef.current`, `start: 'top bottom'`, `end: 'bottom top'`, `scrub: true`.
- The container must have `overflow-hidden` to clip the parallax movement (image is slightly larger than container to accommodate movement — use `scale: 1.15` on the image wrapper via className or inline style).
- Add library ownership comment at top: `// Animation: GSAP ScrollTrigger (scroll-driven parallax — see ANIMATION_LIBRARY_GUIDE.md)`
- Use `loading="eager"` on Image (VFX-05 decision).
- Support `layoutId` prop: if provided, wrap the outer div in `motion.div` with that layoutId (enables lightbox transition later).
- Support `onClick` prop on the container for lightbox trigger.
- Support `children` prop for overlay content (title overlays, gradients).
- Cleanup: `useGSAP` handles cleanup automatically with scope pattern.

**Key constraint:** Maximum 15% movement (intensity capped at 15). If intensity > 15, clamp to 15.

  <action>
Create an `ImageLightbox` component using Framer Motion's `AnimatePresence` and `layoutId` for fullscreen image viewing.

**Props interface:**
```typescript
interface ImageLightboxProps {
  src: string;
  alt: string;
  layoutId: string;           // Must match the triggering element's layoutId
  isOpen: boolean;
  onClose: () => void;
}
```

**Implementation details:**
- Use Framer Motion's `AnimatePresence` wrapping a `motion.div` overlay.
- The overlay: fixed inset-0, z-50, bg-black/90 backdrop with `initial={{ opacity: 0 }}`, `animate={{ opacity: 1 }}`, `exit={{ opacity: 0 }}`.
- Inside the overlay: a `motion.div` with `layoutId={layoutId}` containing the fullscreen image. This creates the seamless origin-to-fullscreen morph transition.
- The image uses Next.js `Image` with `fill` and `object-contain` (not object-cover, since fullscreen should show entire image).
- Close button: absolute top-right, white X icon, `onClick={onClose}`. Use a simple SVG X or Lucide `X` icon.
- Backdrop click: clicking the overlay background (not the image) calls `onClose`.
- Escape key: add `useEffect` with `keydown` listener for Escape key, calling `onClose`. Clean up on unmount.
- Prevent body scroll when open: add `useEffect` that sets `document.body.style.overflow = 'hidden'` when open, restores on close/unmount.
- Add library ownership comment: `// Animation: Framer Motion (layoutId transition + AnimatePresence — see ANIMATION_LIBRARY_GUIDE.md)`
- `AnimatePresence` must have `mode="wait"` for clean enter/exit.
- Transition config: use `layout: { duration: 0.4, ease: [0.25, 0.1, 0.25, 1] }` for the layoutId morph.

**Accessibility:**
- Trap focus inside lightbox when open (at minimum, the close button should be auto-focused).
- `role="dialog"` and `aria-modal="true"` on the overlay.
- `aria-label` on close button.
- Respect `prefers-reduced-motion`: if active, use `duration: 0` for transitions (instant open/close, no morph animation).
