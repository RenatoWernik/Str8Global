Create a `MagneticCursor` component that wraps any interactive element and applies a magnetic pull effect on desktop hover.

**Props interface:**
```typescript
interface MagneticCursorProps {
  children: ReactNode;
  className?: string;
  strength?: number;        // Pull strength 0-1, default 0.3
  threshold?: number;       // Activation distance in px, default 100 (not used if mouse is over element)
  as?: React.ElementType;   // Wrapper element type, default 'div'
}
```

**Implementation details:**
- Use Framer Motion's `useSpring` for x and y position with spring config: `{ stiffness: 150, damping: 15, mass: 0.1 }`.
- Use `useReducedMotion` from `@/hooks/useReducedMotion`.
- Detect mobile with `useState` + `useEffect` checking `window.matchMedia('(max-width: 767px)')`.
- If mobile OR reducedMotion: render children directly with no wrapper effects (POLISH-04).
- On `onMouseMove`: calculate offset from element center, multiply by `strength`, set spring values.
- On `onMouseLeave`: reset spring values to `{ x: 0, y: 0 }` (element returns to original position).
- Wrap children in `motion.div` (or `motion[as]`) with `style={{ x: springX, y: springY }}`.
- The wrapper must NOT change layout — use `display: inline-block` or accept className for flexibility.
- Add library ownership comment: `// Animation: Framer Motion (useSpring magnetic cursor — see ANIMATION_LIBRARY_GUIDE.md)`

**Magnetic effect calculation on mousemove:**
```typescript
const rect = element.getBoundingClientRect();
const centerX = rect.left + rect.width / 2;
const centerY = rect.top + rect.height / 2;
const deltaX = (event.clientX - centerX) * strength;
const deltaY = (event.clientY - centerY) * strength;
springX.set(deltaX);
springY.set(deltaY);
```

**Performance:** Only active when mouse is over the element (onMouseMove/onMouseLeave). No continuous listeners. No requestAnimationFrame loops. Spring physics handled entirely by Framer Motion.
