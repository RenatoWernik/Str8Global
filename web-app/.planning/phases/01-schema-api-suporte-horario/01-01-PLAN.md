<task type="auto">
  <name>Task 2: Update TypeScript types and CRUD with overlap validation</name>
  <files>src/types/database.ts, src/lib/database.ts</files>
  <action>
    **1. Update `src/types/database.ts`:**

    Add `start_time?: string | null` and `end_time?: string | null` to:
    - `ReservationRow` (as optional, after `end_date`)
    - `ReservationInsert` (as optional)
    - `ReservationUpdate` (as optional)

    These are TEXT fields storing "HH:MM" format (e.g. "10:00", "14:00"). NULL means day-based reservation (gear).

    **2. Update `src/lib/database.ts`:**

    a) Add a `checkTimeConflict` helper function:
    ```typescript
    export async function checkTimeConflict(
      itemId: string,
      date: string,
      startTime: string,
      endTime: string,
      excludeReservationId?: string
    ): Promise<boolean>
    ```
    - Query `reservations` where `item_id = itemId`, `status = 'active'`, `start_date <= date`, `end_date >= date`
    - Filter for rows where `start_time IS NOT NULL` (hourly reservations)
    - Check overlap: existing.start_time < endTime AND existing.end_time > startTime
    - If `excludeReservationId` provided, exclude that reservation (for updates)
    - Return `true` if conflict exists, `false` if slot is free

    b) Update `createReservation`:
    - Before inserting, if `reservation.item_type === 'studio'` AND `reservation.start_time` AND `reservation.end_time`:
      - Call `checkTimeConflict(reservation.item_id, reservation.start_date, reservation.start_time, reservation.end_time)`
      - If conflict, throw `new Error('CONFLICT: Time slot overlaps with existing reservation')`
    - Keep existing behavior for gear (no time fields = no overlap check)

    c) Update `updateReservation`:
    - If updates include `start_time` or `end_time`, or if updating `start_date`/`end_date` on a studio reservation:
      - Fetch existing reservation to get full context
      - Run overlap check with `excludeReservationId = id`
      - If conflict, throw same CONFLICT error

    Important: Do NOT change function signatures or return types. The overlap check is added internally before the Supabase insert/update call.
  </action>
  <verify>
    Run `npx tsc --noEmit` from web-app root. Should compile with zero errors. Verify:
    - `ReservationRow` has `start_time` and `end_time` optional fields
    - `checkTimeConflict` function exists and is exported
    - `createReservation` calls `checkTimeConflict` for studio reservations with time fields
  </verify>
  <done>

  </done>
