Create `src/components/animations/RevealImage.tsx` — a reusable image wrapper that reveals images with a GSAP clip-path animation.

**Component interface:**
```tsx
interface RevealImageProps {
  src: string;
  alt: string;
  className?: string;       // Applied to outer container div
  imageClassName?: string;  // Applied to Next.js Image
  aspectRatio?: string;     // Tailwind aspect class e.g. "aspect-[4/5]", default "aspect-video"
  isInView: boolean;        // Controlled by parent section (not self-observed)
  delay?: number;           // Stagger delay in seconds, default 0
  direction?: 'up' | 'down' | 'left' | 'right'; // Reveal direction, default 'up'
}
```

**Implementation details:**
- Use `'use client'` directive
- Add GSAP library ownership comment: `// Animation: GSAP (scroll-triggered clip-path reveal — see ANIMATION_LIBRARY_GUIDE.md)`
- Import `useRef`, `useEffect` from React, `gsap` from 'gsap', `Image` from 'next/image', `useReducedMotion` from `@/hooks/useReducedMotion`
- Container div wraps Image with `overflow-hidden` and `position: relative`
- Initial clip-path state hides the image based on direction:
  - 'up': `inset(100% 0 0 0)` (hidden from bottom, reveals upward)
  - 'down': `inset(0 0 100% 0)` (hidden from top, reveals downward)
  - 'left': `inset(0 100% 0 0)` (hidden from right, reveals leftward)
  - 'right': `inset(0 0 0 100%)` (hidden from left, reveals rightward)
- When `isInView` becomes true AND `prefersReducedMotion` is false:
  - `gsap.fromTo(containerRef.current, { clipPath: initialClipPath }, { clipPath: 'inset(0 0 0 0)', duration: 0.8, delay, ease: 'power3.inOut' })`
- When `prefersReducedMotion` is true:
  - Set clipPath to 'inset(0 0 0 0)' immediately via `gsap.set()` — no animation
- Image component uses `fill`, `className="object-cover"` merged with imageClassName, and `loading="eager"` (VFX-05)
- Use useEffect with deps [isInView, prefersReducedMotion, delay, direction]
- Cleanup: `gsap.killTweensOf(containerRef.current)` in useEffect cleanup
- Do NOT use useGSAP hook (no ScrollTrigger needed — triggered by isInView prop)
- Do NOT create its own IntersectionObserver (VFX-06 — parent controls via isInView prop)
